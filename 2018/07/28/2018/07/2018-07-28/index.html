IE Browser js Fuzzer opensource

前言

这是个半成品项目，js样本生成引擎是我自己写的一个demo,没有参考任何开源js生成引擎，
这个demo是由我实习的时候想添加给公司项目的一个功能——变量传递、联动进一步演变来的。所以
这个js样本生成引擎其实主要功能就是完成了当初我想实现未能实现的功能，变量传递、联动。然
后添加了一些简单的语法，所以生成的代码样式单一（肯定不能跑出crash）。嗯，那这个项目有什
么用呢？我想这个项目虽然不能跑出洞，但是它也基本具备了一个fuzzer具备的功能：样本生成引
擎、崩溃检测程序、Crash记录、多进程fuzz。我想如果有需要学习fuzz的朋友可以通过它来学习
认识fuzz.虽然它很简陋。

变量传递

变量传递，我是通过栈数据结构实现的，同一个作用域内的所有变量，由某一个链表存储，而这个链
表地址存储在一个某一层栈中，上一层栈里存储着作用域更低的一个链表地址，这个链表里同样有着
属于同一作用域的一些变量。当要退出某作用域时，只需要像汇编里的 pop 一样，把对应栈里的链
表内容释放即可。

崩溃检测

崩溃检测，是由createprocess 函数设置为Debug模式来进行获取程序执行的异常码，类似OD能捕
获到各种异常一样。当捕获到栈溢出、访问地址无效、除零异常 等等时就可以判断有一个crash。

多进程

IE 需要设置IE为单进程模式才能进行多进程跑。 网上的教程很多

regedit运行注册表编辑器，依次展开到HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\Main分支，在右侧窗口中查找并双击“TabProcGrowth”（如果没有，请新建一个DWORD32值，并将其命名为TabProcGrowth），在弹出的对话框中输入“0”或者“1”。

使用

把 编译好的 jsfuzz_2018和check_crash_LI 放到 编译好的Crashafl 下 然后运行Crashafl.exe
(Crashafl 只是一个高仿afl的UI程序和afl 没有任何关系)
